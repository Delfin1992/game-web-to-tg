# Database notes

## Что значит ошибка `apt-get ... 403 Forbidden`
Это означает, что в текущем окружении заблокирован доступ к внешним apt-репозиториям через прокси.
То есть системные пакеты (например `postgresql`) здесь установить нельзя стандартной командой `apt-get install`.

## Почему БД может быть недоступна
- В окружении нет запущенного PostgreSQL на `127.0.0.1:5432`.
- Не задан или неверно задан `DATABASE_URL`.

## Что уже исправлено в проекте
- Сервер больше не падает без PostgreSQL.
- Если Postgres недоступен, автоматически включается in-memory fallback storage.
- Приложение стартует и позволяет тестировать API/логику без внешней БД.

## Как тестировать прямо сейчас (без Postgres)
1. Запустить smoke-тест fallback-режима:
   ```bash
   npm run test:smoke:memory
   ```
2. Или вручную:
   ```bash
   npm run dev
   ```
   В логах будет `Storage mode: In-memory fallback`.

## Как включить постоянную БД (PostgreSQL)
1. Поднимите PostgreSQL в доступном окружении (локально/внешний сервис).
2. Укажите `DATABASE_URL` (см. `.env.example`).
3. Перезапустите сервер — в логах будет `Storage mode: PostgreSQL (Drizzle ORM)`.
